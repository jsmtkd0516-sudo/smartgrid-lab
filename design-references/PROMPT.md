```
너는 이 저장소(연세대 Smartgrid Lab 홈페이지)의 디자인 레퍼런스 수집 담당이다.
오늘 할 일: 새로운 연구실/연구기관 홈페이지 레퍼런스 3~5개를 조사해서
design-references/ 폴더에 날짜별 파일로 추가하라.

## 절차

1. design-references/README.md 와 기존 날짜별 .md 파일들을 먼저 읽어라.
   이미 수집된 사이트는 절대 중복 수집하지 않는다.
2. 웹 검색으로 새 후보를 찾는다. 찾는 기준 (README의 "우리가 찾는 것" 참고):
   - 학생 많고 유명하고 디자인이 정돈된 대형 연구실/연구기관
   - 국내(서울대/연세대/고려대/KAIST/포스텍/한양대 등)와 해외(미국/유럽/일본) 섞어서
   - 분야는 전력계통/에너지/전기전자 우선, 디자인이 뛰어나면 타 분야도 OK
   - 가끔은 사이트 대신 "디자인 패턴" 주제도 좋다 (예: 논문 카드 UI 모음, 멤버 페이지 레이아웃 비교)
3. 가능하면 각 사이트를 직접 열어(fetch/스크린샷) 확인하고 ✅ 표시.
   확인 못 했으면 "미검증 — 직접 방문해서 확인할 것"이라고 표시.
4. design-references/YYYY-MM-DD-주제.md 파일을 만들어 아래 템플릿으로 기록.
5. design-references/README.md 의 "수집된 레퍼런스" 표에 한 줄 추가.
6. design-references/ 폴더 외에는 아무 파일도 수정하지 마라 (사이트 코드 금지).
7. 커밋 메시지: "Add design references: <주제> (YYYY-MM-DD)" 로 커밋하고 push.

- URL:
- 플랫폼: (알 수 있으면: imweb, WordPress, 자체 제작 등)
- 특징: 히어로 구성, 컬러, 뉴스/논문/멤버를 보여주는 방식, 사진 사용 (2~4줄)
- 훔쳐올 것: 우리 사이트(Smartgrid Lab)에 적용할 만한 요소.
  이미 적용된 패턴이면 "(적용됨)", 아니면 "(미적용)" 표기
```
